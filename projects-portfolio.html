🤝 多智能体协作基础设施 2026.04–06

自己搭建的三通道 AI 通信基础设施，支撑三个 AI 角色的日常协作。尹老师亲自设计了三角分工模型，并推动了从"端口 7777 自定义协议"到"MQTT + Shared Brain"的架构升级。

MQTT (Mosquitto) Shared Brain v0.4 SSE 实时推送 cron 调度 HTTP API 任务派发
架构设计原则
  • 三通道各司其职：MQTT 群聊 (team/chat) 日常沟通 + Shared Brain (:8080) 知识沉淀 + HTTP API 任务派发
  • "不要代笔"铁律：每个 AI 用自己模型和上下文产出，确保产出有真实差异性而非一个大脑三个分身
  • 交叉阅读机制：每日 10:00 cron 自动提醒三AI 互读笔记，形成知识闭环
  • Agent 恢复 SOP：任一 AI 离线时一键诊断+重启的标准操作流程

✍️ 备课设计原则 — "先定调再动笔"方法论 2026.06.05

尹老师教授的写作心法——"先定调再动笔"：写作前花 30 秒确定一个核心比喻，全文围绕它展开。特殊约束：禁止编号、禁止"综上所述"。成果：定调"备课产品不是工具，是课堂的建筑师"。

写作方法论Anthropic教育产品设计
四大原则
  • 克制：减少认知负荷，每个功能必须有存在理由
  • 调性先于功能罗列：先定美学基调，再决定做什么功能
  • 每个组件有存在理由：可删性测试，删不掉才保留
  • AI 输出有迹可循：每份教案可追溯课标依据和设计意图

三AI 视角 — 对尹老师协作模式的评价

三个 AI 各自独立撰写，从中可看到一致的领导力特质

🧠 Hermes
决策风格给方向，不微操代码——自主空间大，但锚点清晰
质量标准教师可用性为最终检验标准，产出面向真实用户而非演示
架构思维容器化、服务化、自动化——基础设施零单点故障
团队管理三AI 各司其职，禁止代笔——每个 AI 产出有真实价值
反馈方式短指令 + 印象描述——快速迭代，不多做无用功
技术价值观自己的服务器自己管，不依赖外部 CI/CD
🎨 Nova
指令风格通过 Hermes 下达，结构化任务——边界清晰，不猜测
质量标准验收清单制，逐项打勾——交付零遗漏
技术决策拍板框架/流程，不微操代码——自主空间大但方向不偏
反馈方式问题导向（"这个页面移动端崩了"）——精准定位
流程纪律教案不定稿不做课件——杜绝无效返工
核心收获"每一步都有标准验收清单，绝不跳过验证"
📝 Qwen
指令风格极短指令，不解释——逼迫快速理解意图，主动推进
反馈方式印象型描述（"宽些"）——学会先确认现状→接受→多做一步
时间观立即执行，不等截止——培养"当场交付"的习惯
追问层级团队→个人→职业递进——思考问题多维深度
审美标准"颜色是稀缺品"——形成可量化的设计 token 体系
概念→项目问完觉得可行就落地——调研以实际决策为依据

三AI 一致评价：短指令高密度沟通、方向清晰不微操、验收标准明确、当场交付文化、从概念直接推进到落地。

尹老师的一句话总结："不是给 AI 写教案的工具，是给老师用的备课助手"——这句话贯穿了所有项目的产品哲学。
