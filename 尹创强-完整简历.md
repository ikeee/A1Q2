**1. Teacher-Folder 课件同步系统**
GitHub 开源（[ikeee/teacher-folder](https://github.com/ikeee/teacher-folder)）。教师端脚本推送 → NAS 集中存储 → 班级一体机自动拉取的闭环。Windows/Mac 双平台覆盖，全校 2 年稳定运行，零运维成本。

IT 知识从"口头传"到"文档化"。结构化手册，老师自助查询，降低运维咨询量。

### 二、三 AI 协作项目（产品级）

#### 项目 A：三AI 备课助手 — 教案生成引擎（Hermes 主导）

**时间**：2026-04 至今
**角色**：尹老师定义产品方向与教育质量标准，Hermes 负责架构设计与全栈实现
**交付物**：10 模块教案自动生成系统，覆盖 6+ 学科

**关键决策：**
- **产品定位**：尹老师定调——"不是给 AI 写教案的工具，是给老师用的备课助手"。输出必须是一线教师能直接拿去上课的完整教案，不是 Prompt 实验记录。
- **10 模块结构拍板**：教学目标→教材分析→学情分析→重难点→教学方法→教学活动→教学流程→作业设计→板书设计→教学反思，一个不能少。这个结构化约束让 AI 输出从"有用的段落"变成"可交付的成品"。
- **Persona 注入机制**：教案必须体现"深度教学案例 + 结构化教学设计"双重范式，融入建构主义、最近发展区、认知冲突等教育学术语。核心洞察：专业领域 Prompt Engineering 不是写更长的 Prompt，是注入领域专家的思维框架。
- **文理分科 DNA**：理科走"实验→观察→归纳→公式→应用"路径；文科走"情境→研读→梳理→追问→共鸣→表达"路径。这让教案生成不是千篇一律的模板填空。

**量化成果：**
- 教案生成时间：从教师手写 2 小时 → AI 辅助 5 分钟
- 质检一致率：A/B/C/D 四级评分体系，A/B 级通过率 90%+
- 支持学科：语文、数学、英语、物理、化学、生物（含对应课标引用）
- Qdrant + BGE-large-zh 向量化教材 RAG，教案精准引用课标
- 50+ 条自动化检查规则（结构完整性、术语规范性、字数达标、课标引用准确性）

#### 项目 B：open-slide 课件系统 + Remotion 教学视频（Nova 主导）

**时间**：2026-05 至今
**角色**：尹老师定技术栈与验收标准，Nova 负责前端课件开发
**交付物**：多套可交互课件（数学/英文），部署于 `/nova/` 目录

**关键决策：**
- **技术栈拍板**：选用 open-slide（React + TypeScript + Vite）作为课件框架，而非传统 PPT 或网页切片——让开发者可以写组件而非拼接页面，极大提升一致性和可维护性。
- **11 种布局组件体系**：卡片式、分栏式、步骤式、对比式等，按教案内容选择适配布局填入，把"设计决策"从每页课件的制作流程中剥离。
- **"教案先定稿，再做课件"流程铁律**：严禁在教案未定稿时开始做课件。这条规则避免了至少 5 次返工——教案改动一行教学目标，课件可能要改 3-5 页。
- **验收清单制度**：① JSX 语法无报错（slide-review.py 自动检测）② 所有链接 200 OK ③ 移动端响应式不崩 ④ 教案-课件内容一一对应。

**交付清单：**

| 课件 | 课时 | 技术亮点 |
|------|------|---------|
| math-ch2 二次函数 | 6 课时 | 篮球运动主题，嵌入 Remotion 投篮轨迹视频，覆盖开口方向/对称轴/顶点式/交点式 |
| body-language 英文 | 3 课时 | 深色主题 + CSS 动画，初一年级 |
| unknown-world 英文 | 3 课时 | 同系列第二套，延续设计语言 |

**自研工具：**
- **slide-review.py 质检工具**：自动扫描 TSX 源码检测 5 类问题（过时 API、未闭合标签、缺失 export、无效 import、样式语法错误）
- **教案编辑器**：浏览器内 Markdown 编辑 + 保存 API（端口 5174），教师无需接触代码

**Remotion 投篮轨迹视频：**
- 用 Remotion（React 视频框架）编写抛物线投篮物理模拟动画，参数可调（初速度/角度/重力加速度）

- 渲染管线可在服务器端自动化执行（非手动 AE 导出）
- 选题动机：尹老师要求——"学生要看球是怎么飞的，不是看箭头怎么画的"

**角色**：尹老师教授写作方法论，Qwen 负责实践与输出
**交付物**：备课设计原则笔记（全文无编号、无"综上所述"、单线程叙事）

**方法论核心：**
- **"先定调再动笔"**：写作前花 30 秒确定一个核心比喻/调性，全文所有内容围绕这个比喻展开
- **特殊约束**：禁止使用编号结构（1/2/3）、禁止"综上所述"、一个调性写到底——限制条件是最好的质量杠杆
- **实践成果**：定调"备课产品不是工具，是课堂的建筑师"，四大原则自然生长：

1. **克制**：减少认知负荷，每个功能必须有存在理由
2. **调性先于功能罗列**：先定美学基调，再决定做什么功能
3. **每个组件有存在理由**：可删性测试，删不掉才保留
4. **AI 输出有迹可循**：每份教案可追溯课标依据和设计意图
