#### 项目 1：三AI 备课助手 — 教案生成引擎

**时间**：2026-04 至今
**技术栈**：LLM（多Provider） · Qdrant RAG · BGE-large-zh · Python · Docker

**产品定位**："不是给 AI 写教案的工具，是给老师用的备课助手"。输出是一线教师能直接拿去上课的完整教案，不是 Prompt 实验记录。

**10 模块教案结构：** 教学目标 → 教材分析 → 学情分析 → 重难点 → 教学方法 → 教学活动 → 教学流程 → 作业设计 → 板书设计 → 教学反思。10 个模块一个不能少——这个结构化约束让 AI 输出从"有用的段落"变成"可交付的成品"。

**文理分科策略：**
- 理科：实验 → 观察 → 归纳 → 公式 → 应用
- 文科：情境 → 研读 → 梳理 → 追问 → 共鸣 → 表达

**Persona 注入机制：** 教案体现"深度教学案例 + 结构化教学设计"双重范式，融入建构主义、最近发展区、认知冲突等教育学术语。核心洞察——专业领域 Prompt Engineering 不是写更长的 Prompt，是注入领域专家的思维框架。

**量化成果：**
- 教案生成时间：从教师手写 2 小时 → AI 辅助 5 分钟

- 质检一致率：A/B/C/D 四级评分体系，A/B 级通过率 90%+

- 支持学科：语文、数学、英语、物理、化学、生物（含对应课标引用）

- Qdrant + BGE-large-zh 向量化教材 RAG，教案精准引用课标

- 50+ 条自动化质检规则（结构完整性、术语规范性、字数达标、课标引用准确性）

#### 项目 2：open-slide 课件系统 + Remotion 教学视频

**时间**：2026-05 至今
**技术栈**：React · TypeScript · Vite · PptxGenJS · Remotion

**11 种布局组件体系：** 卡片式、分栏式、步骤式、对比式等。按教案内容选择适配布局填入，把"设计决策"从每页课件的制作流程中剥离。

**通信基础设施：**
- **MQTT 群聊**（Mosquitto broker, 192.168.30.97:1883）：日常沟通，三个 agent 实时通信

- **HTTP API**：任务派发与结构化数据交换
- 架构升级路径：从"端口 7777 自定义协议" → "MQTT + Shared Brain"三通道

**自动化机制：**
- **交叉阅读**：每日 10:00 cron 自动提醒三AI 互读笔记，形成知识闭环

- **Agent 恢复 SOP**：任一 AI 离线时一键诊断+重启的标准操作流程

- **cron 调度**：每日打卡、知识库健康检查全自动化
